# Аналитика маркетплейса: от SQL до Power BI

> Дипломный проект по анализу данных e-commerce платформы  
> **Автор:** Екатерина  
> **Дата:** Июль 2026

## Цель проекта

Создание системы мониторинга ключевых метрик (KPI) и проведение глубокого исследования эффективности бизнеса:
1. **Операционный дашборд** — мониторинг продаж в реальном времени
2. **Оптимизация ассортиментной матрицы** — ABC-анализ, поиск неликвида
3. **Повышение LTV клиентской базы** — RFM-сегментация, анализ оттока

Исследование 1: Оптимизация ассортиментной матрицы
Использованные методы:
ABC-анализ по выручке
Scatter Plot "Выручка vs Давность последних продаж"
Ранжирование товаров по принципу Парето
Ключевые выводы:
Топ-10 товаров генерируют ~45% всей выручки (принцип Парето 80/20)
Выявлена группа "мёртвого груза" (15% ассортимента):
Выручка < 0.1% от общей
Давность последней продажи > 90 дней
Низкая оборачиваемость склада
На графике "Мертвый товар" (справа внизу) отчётливо видны товары-аутсайдеры, которые:
Практически не приносят выручки
Не продавались более 3 месяцев
Замораживают оборотные средства

Рекомендации для бизнеса:
Приоритет	     Действие	                                                                            Ожидаемый эффект
Высокий	       Вывести из ассортимента товары с `days_since_last_sale > 90` и `revenue < 10 000 ₽`	 Высвобождение ~X ₽ оборотных средств
Средний	       Распродажа со скидкой 30-50% для товаров группы C	                                   Ускорение оборачиваемости, возврат части средств
Низкий	        Увеличить сток для Топ-20 товаров (группа A)	                                        Стабильный cash flow, отсутствие out-of-stock
Доп.	          Cross-sell акции для товаров группы B: "Купи этот товар — получи скидку на лидер"	   Подтягивание группы B к лидерам


Метрики успеха:
Увеличение оборачиваемости склада с X дней до Y дней
Рост доли выручки от Топ-20 товаров с 45% до 60%
Снижение доли неликвида с 15% до 5%


Исследование 2: Клиентская база и повышение LTV
Использованные методы:
RFM-сегментация (Recency, Frequency, Monetary)
Анализ когорт и удержания
Churn Rate по интервалам 30 дней
Scatter Plot "Частота покупок vs Сумма покупок (LTV)"


Ключевые выводы:
 Критическая точка оттока: Пик приходится на интервал 90-120 дней без покупок
 VIP-сегмент (5% клиентов) генерирует ~35% всей выручки
 На графике "Анализ оттока" видна классическая кривая:
0-100 дней (слева): активные покупатели — самый высокий столбец
100-400 дней (центр): клиенты под риском ухода
400+ дней (справа): практически потерянные клиенты — длинный "хвост"
 Сегментация клиентов (Scatter Plot):
Правый верхний угол: VIP-клиенты (много покупают, часто)
Левый верхний угол: Крупные разовые покупатели (высокий LTV, низкая частота)
Правый нижний угол: Частые, но мелкие покупки
Левый нижний угол: Новые или "спящие" клиенты
Рекомендации для бизнеса:

Приоритет	   Действие	                                                                                    Целевая аудитория	                     Ожидаемый эффект
Высокий 	    Триггерная email/SMS рассылка на 90-й день без покупки с промокодом "RETURN15" (-15%)	       Клиенты 90-120 дней без покупок	        Возврат 10-15% "засыпающих" клиентов
Средний	     Программа лояльности: 1% кэшбэка за каждую покупку с прогрессивной шкалой	                   Все клиенты, особенно первые 6 месяцев	 Увеличение частоты покупок на 20-25%
Низкий	      Персональные предложения для VIP-сегмента: ранний доступ к новинкам, персональный менеджер	  Топ-5% по LTV	                          Удержание VIP, рост среднего чека на 15-20%
Доп.	        Реактивационная кампания для клиентов 400+ дней: "Мы скучаем, вернись со скидкой 25%"	       Потерянные клиенты	                     Возврат 3-5% клиентов


Метрики успеха:
Снижение оттока (Churn Rate) с X% до Y% за 6 месяцев
Рост среднего LTV клиента с A ₽ до B ₽
Увеличение доли повторных покупок с C% до D%
Рост NPS (Net Promoter Score) на 10-15 пунктов
